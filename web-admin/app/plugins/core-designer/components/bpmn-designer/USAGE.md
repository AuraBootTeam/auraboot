# BPMN设计器使用指南

## 🎯 新功能使用说明

### 1. 移动节点位置

**功能说明**: 在画布中自由拖动节点调整布局

**使用方法**:
1. 将节点从左侧组件库拖放到画布
2. 点击并按住画布中的任意节点
3. 拖动鼠标移动节点到新位置
4. 松开鼠标完成移动

**特性**:
- ✅ 自动对齐网格（20px间距）
- ✅ 连接线自动跟随节点移动
- ✅ 移动后自动标记为未保存状态

**示例**:
```
1. 拖放"开始事件"到画布
2. 拖放"用户任务"到画布
3. 点击并拖动"用户任务"调整位置
4. 节点之间的连线会自动调整
```

---

### 2. 编辑连线名称

**功能说明**: 为流程连线添加说明文字，例如"同意"、"拒绝"、"金额>1000"等

**使用方法**:

#### 步骤1: 创建连线
1. 从一个节点的右侧连接点拖动到另一个节点
2. 连线创建后会显示在画布上

#### 步骤2: 编辑连线名称
1. 点击连线选中它（连线会变蓝色高亮）
2. 右侧属性面板会显示"连线属性"
3. 在"连线标签"输入框中输入名称
4. 标签会立即显示在连线上

**标签样式**:
- 字体颜色: 深灰色 (#374151)
- 字体大小: 12px
- 背景: 白色半透明
- 圆角边框: 4px
- 内边距: 8px x 4px

**使用场景示例**:

**场景1: 审批流程**
```
开始 → [提交申请] → 部门经理审批
                      ↓ (同意)
                    总经理审批
                      ↓ (同意)
                    结束
```

连线标签:
- "提交申请"
- "同意"
- "拒绝"

**场景2: 金额判断**
```
开始 → [金额判断网关]
         ↓ (金额 > 1000)
       经理审批
         ↓ (金额 ≤ 1000)
       主管审批
```

连线标签:
- "金额 > 1000"
- "金额 ≤ 1000"

## 📋 完整操作流程示例

### 创建请假审批流程

**步骤1: 设置流程信息**
```
流程名称: 员工请假审批流程
流程标识: leave_approval_process
```

**步骤2: 添加节点**
1. 拖放"开始事件"到画布左侧
2. 拖放"用户任务"到中间，命名为"提交请假申请"
3. 拖放"用户任务"到右侧，命名为"部门经理审批"
4. 拖放"结束事件"到最右侧

**步骤3: 调整节点位置**
- 将节点横向排列，间距约200px
- 确保节点对齐

**步骤4: 创建并命名连线**
1. 连接: 开始 → 提交请假申请
   - 标签: "发起申请"

2. 连接: 提交请假申请 → 部门经理审批
   - 标签: "提交审批"

3. 连接: 部门经理审批 → 结束
   - 标签: "审批完成"

**步骤5: 配置人员分配**

点击"提交请假申请"节点:
- 人员分配类型: 流程发起人
- 审批模式: 单人审批

点击"部门经理审批"节点:
- 人员分配类型: 指定角色
- 角色ID: dept_manager
- 审批模式: 单人审批
- 优先级: 80

**步骤6: 验证并保存**
1. 点击顶部"验证"按钮
2. 确认无错误
3. 点击"保存"按钮

## 🎨 界面说明

### 画布操作

**缩放**:
- 鼠标滚轮: 缩放画布
- Ctrl + 滚轮: 更精细的缩放
- 右下角控制按钮: +/- 缩放

**平移**:
- 按住空白区域拖动: 移动画布视图
- 右下角控制按钮: 居中/适应视图

**小地图**:
- 右下角显示整体布局
- 点击小地图可快速定位

### 选择状态指示

**节点选中**:
- 蓝色边框高亮
- 右侧显示节点属性面板

**连线选中**:
- 蓝色线条高亮（加粗）
- 右侧显示连线属性面板

**未保存标识**:
- 顶部显示"未保存"橙色标签
- 移动节点、编辑连线、修改属性都会触发

## 💡 最佳实践

### 1. 命名规范

**节点命名**:
- 使用动词开头: "审批请假"、"发送通知"
- 简洁明了: 不超过10个字
- 体现业务含义

**连线标签**:
- 条件判断: "同意"、"拒绝"、"金额>1000"
- 流转说明: "提交审批"、"退回修改"
- 可以为空: 默认流向不需要标签

### 2. 布局建议

**横向布局**:
```
[开始] → [任务1] → [任务2] → [任务3] → [结束]
```
适合: 线性流程、顺序审批

**纵向布局**:
```
[开始]
  ↓
[网关]
  ↓ (条件1)
[任务1]
  ↓ (条件2)
[任务2]
  ↓
[结束]
```
适合: 分支流程、条件判断

**混合布局**:
根据实际业务逻辑自由组合

### 3. 连线标签使用场景

**必须添加标签**:
- ✅ 从网关出发的所有连线
- ✅ 有条件判断的连线
- ✅ 需要说明用途的连线

**可选添加标签**:
- ⚪ 默认的顺序流转
- ⚪ 单一的流向
- ⚪ 从任务到结束事件

## ⚠️ 注意事项

### 节点移动
1. 移动节点后记得保存
2. 多次移动会叠加"未保存"状态
3. 网格对齐帮助保持整洁布局

### 连线编辑
1. 必须先点击选中连线才能编辑
2. 连线标签支持中英文、数字、符号
3. 标签过长会自动换行
4. 空标签不会显示在连线上

### 性能建议
- 节点数量建议: < 50个
- 连线数量建议: < 100条
- 复杂流程建议使用子流程拆分

### 3. 导入流程定义文件

**功能说明**: 从本地JSON文件导入流程定义，快速加载已保存的流程

**使用方法**:
1. 点击顶部工具栏的 `📥 导入` 按钮
2. 在文件选择对话框中选择 `.json` 格式的流程定义文件
3. 点击"打开"
4. 系统自动导入节点和连线到画布
5. 看到"导入成功！"提示

**导入效果**:
- ✅ 所有节点和连线显示在画布上
- ✅ 流程名称和标识自动填充到工具栏
- ✅ 支持version版本字段
- ✅ 自动清理React Flow内部字段
- ✅ 自动标记为"未保存"状态

**注意事项**:
- ⚠️ 导入会**完全替换**当前画布内容
- ⚠️ 导入前建议先导出当前流程作为备份
- ⚠️ 仅支持 `.json` 格式文件

**示例**:
```
场景: 导入请假审批流程

1. 准备文件: leave_process.json
2. 点击"📥 导入"按钮
3. 选择文件并打开
4. 画布显示完整的请假审批流程（5个节点，5条连线）
5. 工具栏显示"新建流程"和"process_xxx"
```

详细文档: 查看 `IMPORT_EXPORT.md`

### 4. 导出流程定义文件

**功能说明**: 将当前设计的流程导出为JSON文件保存到本地

**使用方法**:
1. 点击顶部工具栏的 `📤 导出` 按钮
2. 浏览器自动下载JSON文件
3. 文件名格式：`{流程标识}.json`

**导出内容**:
- ✅ 所有节点和位置信息
- ✅ 所有连线和标签
- ✅ 流程元数据（名称、标识、版本等）
- ✅ 节点配置（人员分配、服务配置等）

**使用场景**:
- 💾 备份流程设计
- 📤 分享给团队成员
- 🔄 跨环境迁移流程
- 📋 保存常用流程模板

### 5. 删除节点和连线

**功能说明**: 删除画布中选中的节点或连线

**使用方法**:

#### 方法1: 使用键盘快捷键
1. 点击选中要删除的节点或连线
2. 按 `Delete` 或 `Backspace` 键
3. 节点/连线立即被删除

**注意**: 在输入框中输入文字时，Delete/Backspace键不会触发删除功能

#### 方法2: 使用删除按钮
1. 点击选中要删除的节点或连线
2. 右侧属性面板会显示🗑️删除按钮
3. 点击删除按钮
4. 在确认对话框中点击"确定"

**删除效果**:
- ✅ 删除节点时，所有连接到该节点的连线也会自动删除
- ✅ 删除后自动标记为未保存状态
- ✅ 删除操作即时生效，无需额外保存

**示例**:
```
场景: 删除审批流程中的某个任务节点

1. 点击选中"部门经理审批"节点
2. 按Delete键 或 点击属性面板的删除按钮
3. 节点及其所有连线被删除
4. 顶部显示"未保存"提示
```

## 🆘 常见问题

**Q: 节点无法拖动怎么办?**
A: 确保点击的是节点本身，而不是连接点。连接点用于创建连线。

**Q: 连线标签不显示?**
A: 检查是否输入了标签文字，空标签不会显示。刷新页面后标签会正确显示。

**Q: 如何删除连线?**
A: 选中连线后按Delete或Backspace键，或点击右侧属性面板中的🗑️删除按钮。

**Q: 保存后连线标签消失?**
A: 这是已知问题，请确保在保存前所有标签都已正确输入，刷新后会恢复。

**Q: 如何导出流程图?**
A: 点击顶部"导出"按钮，会下载JSON格式的流程定义文件。
