## AuraBoot v0.1.0-beta.1 — first public beta

Released {date}.

This is the first public beta of AuraBoot — an open-source low-code platform
where you write declarative JSON instead of CRUD boilerplate, with AI built into
the core. We've spent {N months} on it; it's not finished, but it's enough to
build real business apps and we'd rather get it in your hands than keep polishing
in private. Honest critique very welcome.

### Highlights

- 🎯 **DSL Engine** — define a model in JSON and get the Postgres table, REST
  CRUD, list/detail pages, and audit log without writing a line of code.
- 🎯 **20-stage command pipeline** — every mutation flows through a configurable
  pipeline (validate → permission → state machine → handler → side effects → audit).
  Per-command DSL skips, replaces, or reorders stages.
- 🎯 **AI in the core** — AuraBot in-app copilot, Agent Control Plane,
  ChatBI (NL → SQL → chart), RAG knowledge base. Multi-LLM via a unified
  provider layer (OpenAI, Anthropic, Zhipu, Ollama, MiniMax).
- 🎯 **BPMN 2.0 workflows** — SmartEngine-based human approval flows with SLA
  tracking, escalation, and a visual designer. Real human-in-the-loop, not
  if-this-then-that triggers.
- 🎯 **Plugin system** — PF4J backend + Module Federation frontend. 20+
  first-party plugins ship as JSON config-only packages. Adding a new module =
  `plugin.json` + a few resources.

### What's new in this release

This is the first numbered release; the full feature set is the highlights above,
plus the launch-readiness work below.

#### ✨ Added

- DemoModeGuard servlet filter (`aura.demo.enabled=true`) for protected
  staging / demo deployments — rejects destructive endpoints with HTTP 403.
- Per-tenant row scope at the SQL injection layer (no possibility of forgotten
  `WHERE tenant_id`).
- Multi-arch Docker images on GHCR (`linux/amd64`, `linux/arm64`).
- 20-stage pipeline observability with per-stage timing.
- Plugin hot-reload during development.
- License signing with `kid` header support for keypair rotation.

#### ♻️ Changed

- License is **AuraBoot License v1.1** — Apache 2.0 + supplementary terms.
  Free for self-hosting / ISV / vertical SaaS. Commercial license needed only
  for hosting AuraBoot itself as a low-code SaaS service.
- README, FAQ, NOTICE, CLA, CHANGELOG, TELEMETRY all in place at canonical paths.

#### 🐛 Fixed

- HttpMessageNotReadableException now returns 400 instead of 500 (#110).
- AuraBot stream filter correctly buffers partial `<think>` open tags (#108).

#### 🔒 Security

- Historical leaked `license/private.pem` purged from git history via
  `git filter-repo` (committed `2026-05-09`). All clones from before this date
  must reclone. New keypair generation runbook in
  `scripts/dev/rotate-license-keypair.sh`.
- gitleaks pre-commit + CI scan baseline established. Zero leaks at release.

### Known issues

- Mobile UI lags desktop in feature coverage by ~one minor version. A public
  mobile roadmap doc lands with v0.2.x; until then, mobile gaps live in
  GitHub Issues with the `area: frontend` + `mobile` labels.
- Page Designer UX is functional, not delightful. Improving in 0.2.x.
- No managed cloud offering yet. ETA M6+.
- Performance baseline benchmark suite is not yet public. ETA M2.

### Upgrade

Fresh install (only path supported for beta.1):

```bash
git clone https://github.com/AuraBootTeam/auraboot.git
cd auraboot
docker compose up -d
```

### Backward compatibility

Beta-tier release — minor versions during 0.x may include backward-incompatible
changes (called out in `### Breaking changes`). Patch versions never break.
