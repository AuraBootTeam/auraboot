# Internal docs

These directories are working notes for AuraBoot contributors — not part of the public documentation surface.

- **`backlog/`** — open items, follow-ups, and known gaps. Each file records what is known to be incomplete and who is on the hook.
- **`handover/`** — session handover notes capturing in-progress context across days or contributors.
- **`plans/`** — implementation plans and design notes ahead of execution.

**Public readers** should not treat anything here as authoritative documentation:
- For product positioning and architecture, see [`docs/positioning.md`](../positioning.md) and [`ARCHITECTURE.md`](../../ARCHITECTURE.md).
- For end-user docs, see [docs.auraboot.com](https://docs.auraboot.com).

These files may reference internal tickets, private repositories, or transient state. They are checked in for contributor continuity, not for external consumption.
