把菜单默认入口改成工作台
父级不应落到 rollout 列表，也不应叫“预览”。
复用已有条件构建器
ConditionBuilder 是内核，缺的是产品壳和统一任务流。
工程判断：不直接推翻重写，也不能只在旧页面上打补丁
当前问题不是单个组件没做，而是入口和任务流割裂。建议新增一个统一工作台壳层，复用旧的条件、绑定、测试、影响分析能力。

新写的部分

  • RuleCenterWorkbench：规则中心默认入口
  • RuleConfiguratorWorkspace：条件、绑定、映射、测试、影响、发布的一体化壳
  • 业务场景入口协议：SLA / BPM / Permission / Event Policy 统一跳转

复用的部分

  • ConditionBuilder：条件组、字段、操作符、值编辑
  • DecisionRuleBindingBlock：决策绑定、输入输出映射、摘要态
  • PolicyRulesEditor / EventPolicyDesignerBlock：策略编辑经验和测试能力

不建议做的事

  • 不把 SLA、BPM、规则中心各自做一套条件 UI
  • 不把主界面做成 JSON / trace / DMN raw 编辑器
  • 不把 rollout 列表继续作为决策中心默认页
