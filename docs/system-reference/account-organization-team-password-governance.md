# 账号、组织、团队与密码治理

本文定义 AuraBoot 在 SaaS 场景下账号、组织、团队、密码策略的产品边界和默认行为。目标是避免“菜单存在但领域关系不清”“策略配置和实际行为不一致”“客户交付时基础账号闭环缺失”。

## 领域模型

| 概念 | 定位 | 关键职责 | 关系 |
| --- | --- | --- | --- |
| 用户账号 `ab_user` | 全局身份 | 登录凭据、手机号、邮箱、头像、密码、安全状态 | 一个用户账号可以加入多个租户 |
| 租户成员 `ab_tenant_member` | 某租户下的账号身份 | 成员状态、租户角色、租户权限、管理员重置密码 | 绑定一个全局用户账号，属于一个租户 |
| 组织架构 `org_department` | 组织树 | 部门、层级、上下级归属 | 人员归属组织架构 |
| 职位 `org_position` | 岗位字典 | 职位、职级、岗位属性 | 人员可引用职位 |
| 人员 `org_employee` | 组织内人员档案 | 姓名、工号、部门、职位、汇报关系、联系方式、在职/离职状态 | 可绑定租户成员；也可以先作为非登录人员存在 |
| 团队 `org_team` | 协作分组 | 跨部门项目组、临时组、虚拟团队 | 和组织架构平行，引用已有成员或人员 |

核心原则：

- `用户账号` 是全局登录主体，不属于某一个租户。
- `租户成员/账号` 是用户账号在某个租户里的身份和权限载体。
- `人员` 是组织人事档案，不等于登录账号。人员可以先创建，后续再开通账号。
- `团队` 不是组织树的替代品。团队用于跨部门协作，和组织架构平行。
- `离职` 是租户内人员和成员生命周期动作，不是全局用户删除动作。一个用户在 A 租户离职，不能影响他在 B 租户的成员身份。
- `公开注册` 默认关闭。企业 SaaS 默认由租户管理员托管成员来源，不能允许匿名用户自由注册进租户。

账号和人员的关系必须保持松耦合：

- `人员` 可以没有登录账号，用于先维护组织通讯录、部门、职位、汇报关系和在离职状态。
- `账号/租户成员` 必须对应一个全局 `ab_user`，用于登录、授权、会话和密码治理。
- `人员开通账号` 是从人事档案进入身份系统的动作，不是创建人员的必填前置条件。
- `账号绑定人员` 是把已有登录身份挂到组织档案的动作，用于避免人员档案和登录身份割裂。

由此得到两个设计约束：

- 组织人员档案不能因为还没有登录账号而无法创建，否则客户无法先维护真实组织架构。
- 关闭公开注册以后，系统必须保留受控成员来源；不能把“禁止匿名注册”误实现成“租户管理员也无法新增成员”。

## 菜单信息架构

`组织管理` 承载客户交付时必须可见的组织和账号基础能力：

1. `组织架构`：维护部门树。
2. `职位`：维护岗位/职位字典。
3. `人员`：维护组织人员档案，支持新增人员、离职处理、绑定/开通登录账号。
4. `团队`：维护跨部门协作组。
5. `账号`：维护租户成员、角色归属、账号状态、成员导入、管理员重置密码；当前已支持从账号页选择已有人员并开通租户成员。
6. `角色`：维护租户角色。
7. `权限/授权关系`：维护权限授权视图。

`系统管理` 承载部署级和平台级配置：

- 登录渠道、系统偏好、插件管理等平台配置。
- 模型服务配置，例如 LLM provider。
- 账号安全策略页放在系统管理下，当前以只读方式展示部署级密码规则和默认账号行为，避免客户误以为密码规则已支持租户级覆盖。
- 注册策略后续可在系统管理或租户设置下扩展，但必须区分“部署级密码规则”和“租户级行为开关”。

## 密码与账号安全策略

当前采用“部署级密码规则 + 租户级账号行为”的折中模型，避免多租户用户跨租户时出现密码规则冲突。

部署级密码规则保持全局一致：

- 密码复杂度：长度、大小写、数字、特殊字符要求。
- 历史密码：禁止复用最近 N 次密码。
- 密码过期天数。
- 登录失败锁定阈值和锁定时间。
- reset token 有效期。

租户级账号行为只控制高价值开关：

- 登录方式：密码、短信、邮箱验证码、SSO。
- 自助密码：是否允许忘记密码、token 重置、个人改密。
- 注册方式：公开注册、邀请注册、管理员新增、SSO 同步。默认关闭公开注册。
- 管理员托管：是否允许管理员重置成员密码。
- 首次登录或重置后是否强制改密。
- 后续可扩展 MFA、SSO 强制策略、会话策略。

默认策略：

- 默认是管理员托管：自助密码关闭。
- 默认关闭公开注册：登录页不展示注册入口，匿名注册 API 应拒绝。
- 新成员默认只能通过管理员受控入口进入，例如成员导入、人员开通账号、账号页从人员开通成员、邀请链接、SSO 同步；其中成员导入、人员开通账号、账号页从人员开通成员属于当前已闭环入口，邀请和 SSO 同步属于后续增强入口。
- 未登录找回密码不按租户切策略；如果自助恢复关闭，未登录恢复入口应隐藏或拒绝。
- 登录后个人改密、管理员重置密码等有明确租户上下文的动作，才能读取租户级行为策略。

注册和找回密码的边界：

- 匿名公开注册不是默认成员来源。首个管理员、租户 owner、测试账号初始化都属于部署初始化或后台运维流程，不归入公开注册。
- 未登录找回密码没有可靠租户上下文，因此当前默认沿用部署级自助密码开关；不开启自助密码时入口隐藏或 API 拒绝。
- 租户级策略只在已知当前租户时生效，例如登录后的个人改密、租户管理员重置成员密码、租户内成员状态变更。

## 注册关闭后的成员来源

企业 SaaS 默认关闭公开注册，但不等于关闭开户。成员来源按控制强度分层：

| 来源 | 面向对象 | 当前定位 | 是否默认启用 |
| --- | --- | --- | --- |
| 部署初始化 | 首个系统管理员、租户 owner、测试账号 | 运维/初始化流程，不是公开注册 | 是 |
| 仅建人员档案 | 没有登录需求的组织人员 | 先维护组织通讯录、部门、职位、汇报关系 | 是 |
| 租户成员导入 | 已确定要加入租户的成员账号 | 批量创建或维护 `ab_tenant_member` | 是，受权限控制 |
| 管理员重置密码 | 已是租户成员的账号 | 管理员托管密码，返回一次性临时密码 | 是，受权限控制 |
| 人员页开通账号 | 已有人员档案，需要登录系统 | 创建/复用 `ab_user`、创建/复用 `ab_tenant_member`、绑定人员并返回一次性临时密码 | 是，受权限控制 |
| 账号页从已有人员开通成员 | 管理员在账号列表直接从未绑定人员开户 | 选择已有 `org_employee`，复用人员开通账号逻辑，创建/复用用户和成员并返回临时密码 | 是，受权限控制 |
| 账号页完整新增成员并绑定人员 | 管理员在账号页同时新建人员、创建/复用用户、分配角色 | 一站式新增成员、绑定人员、分配角色、生成临时密码或邀请 | 后续增强 |
| 邀请注册 | 管理员邀请外部用户补全资料 | 受控注册，不是匿名公开注册 | 后续增强 |
| SSO/IdP 同步 | 企业身份源同步成员 | 外部身份治理主导 | 后续增强 |

当前交付口径：

- 默认关闭普通匿名注册，防止未知用户自行进入租户。
- 关闭注册只关闭“匿名自助进入租户”，不关闭管理员受控开户。管理员仍可通过人员档案、成员导入、账号维护等入口把确定的成员加入租户。
- 组织人员可以先录入，不要求绑定登录账号。
- 租户成员列表支持成员查询、状态流转、导入、移除、管理员重置密码。
- 人员列表支持从已有人员行发起“开通账号”，完成用户、租户成员和人员档案绑定；如果新建用户，会一次性展示临时密码。
- 账号页从已有人员开通成员已闭环。它复用 `人员 -> 开通账号` 的后端语义，但入口在账号页，便于管理员从成员维护视角完成日常开户。
- 账号页“一站式新增人员 + 创建/复用用户 + 分配角色 + 开通成员”的完整向导仍是后续增强，不能和本迭代的“从已有人员开户”混为一谈。

关闭注册后的场景闭环按以下规则判断：

| 场景 | 是否允许 | 正确入口 | 说明 |
| --- | --- | --- | --- |
| 未知匿名用户自己注册进租户 | 默认不允许 | 无 | 登录页不展示注册入口，匿名注册 API 默认拒绝 |
| 管理员先维护组织档案 | 允许 | `组织管理 -> 人员` | 只建人员，不要求登录账号 |
| 管理员给已有人员开通登录 | 允许 | `人员 -> 开通账号` | 当前已闭环，返回一次性临时密码 |
| 管理员在账号页从人员开户 | 允许 | `组织管理 -> 账号 -> 从人员开通账号` | 已完成配置导入、权限绑定、E2E 和截图验证 |
| 批量导入成员 | 允许 | `组织管理 -> 账号 -> 导入` | 受权限控制，用于批量开户或成员维护 |
| 管理员邀请用户补全资料 | 后续 | 邀请注册 | 受控注册，不是公开注册 |
| 用户申请加入租户 | 后续 | 审批注册 | 需要审批流、租户选择和风控 |
| 企业身份源同步 | 后续 | SSO/IdP | 由外部身份源治理成员生命周期 |

## 关键流程

### 登录与登出

登录成功后返回当前账号和租户身份；如果密码过期，响应中带 `mustChangePassword=true`。

登出采用业界常规方案：服务端撤销当前 session/token，同时前端清理 BFF cookie 和本地会话状态。这样可以覆盖同一 token 被继续使用的风险，也不会把登出变成纯前端清理。

### 用户资料编辑

用户资料编辑归属于全局用户账号，包含基础资料和头像上传。用户跨多个租户时，头像、手机号、邮箱等全局资料保持一致；租户内昵称、角色、状态等租户属性归属于租户成员。

### 管理员重置密码

管理员重置密码必须满足闭环：

1. 管理员在租户成员列表或用户管理入口发起重置。
2. 后端校验管理员权限和目标成员归属。
3. 后端生成符合部署级复杂度策略的临时密码。
4. 返回临时密码给前端一次性展示，并复制到剪贴板。
5. 前端明确提示“临时密码只显示一次”。

当前默认不强制 `mustChangePassword`。后续如果租户级策略开启“重置后强制改密”，管理员重置应同时标记用户下次登录必须改密。

### 新增人员与开通账号

推荐支持三种成员来源：

- 仅创建人员档案：生成 `org_employee`，用于组织结构、汇报关系、通讯录，不创建登录账号。
- 人员页开通账号：从已有 `org_employee` 创建或复用全局 `ab_user`，再创建或复用当前租户 `ab_tenant_member`，并绑定双方关系。
- 账号页从已有人员开通成员：管理员在成员列表选择一个未绑定账号的 `org_employee`，复用人员开通账号逻辑创建或复用全局 `ab_user` 和当前租户 `ab_tenant_member`，绑定人员与成员，并在新建用户时一次性展示临时密码。
- 账号页完整新增成员：目标是创建或复用全局 `ab_user`，创建当前租户 `ab_tenant_member`，并可同时创建/绑定 `org_employee`、部门、职位、团队、角色。该完整向导仍属后续增强。

团队页不作为主开人入口。团队只添加已有成员或人员；新增人员应该在 `人员` 或 `账号` 流程内完成。

当前默认要求：

- 创建人员时 `系统用户/登录账号` 不应是必填项。
- 创建租户成员时必须有登录账号，因为成员承载租户权限和登录后的身份状态。
- 人员离职、成员停用、团队移除是三个不同动作，不能互相替代。
- 当前交付演示可以使用已验证的人员档案、人员开通账号、账号页从人员开通成员、租户成员列表、成员导入、成员状态、管理员重置密码路径。
- 账号页从已有人员开通成员已具备页面入口、人员选择、权限、配置导入、E2E 和截图证据。
- 账号页完整新增并绑定人员仍属后续增强。

### 成员离职与账号停用

离职必须避免误伤跨租户账号：

1. 将当前租户下的 `org_employee` 标记为离职或停用。
2. 将当前租户下的 `ab_tenant_member` 标记为 disabled、resigned 或等价状态。
3. 撤销该成员在当前租户的活动 session/token。
4. 保留全局 `ab_user`，除非确认该用户在所有租户都不再可用。
5. 团队、角色、组织视图应能识别离职/停用状态，避免继续作为有效协作者使用。

物理删除只适用于误建数据或合规清理，不能作为默认离职动作。

当前实现采用保守闭环：

- 租户成员 `inactive` 时，同步把已绑定人员标记为离职。
- 租户成员 `inactive` 或 `suspended` 时，撤销该用户现有 session。
- 不禁用或删除全局 `ab_user`，避免影响同一个用户在其他租户的身份。
- 因现有 session 表未显式租户分区，当前撤销粒度是该用户现有 session；后续如引入租户级 session，应收敛为当前租户 session。

### 注册入口

默认关闭公开注册：

- 登录页不展示普通注册入口。
- 匿名注册 API 默认拒绝。
- 首个系统管理员或租户 owner 的创建属于部署初始化或后台开通流程，不属于公开注册。

未来如果客户需要注册能力，应按租户显式开启，并优先采用受控方式：

- 邀请注册：管理员生成邀请，用户通过邀请链接补全资料。
- 域名白名单：企业邮箱域名匹配后允许加入指定租户。
- 审批注册：用户申请加入租户，管理员审批后成为成员。
- SSO 同步：由企业 IdP 同步成员和登录身份。

公开注册、邀请注册、审批注册、SSO 同步需要和租户级账号行为策略绑定。未配置策略页前，默认行为保持保守：公开注册关闭，自助密码关闭，管理员托管开启。

## 当前实现状态

已闭环的行为：

- 组织管理菜单已恢复 `组织架构 / 职位 / 人员 / 团队 / 账号 / 角色 / 权限/授权关系`。
- 系统管理菜单已补充 `模型服务` 入口。
- 当前会话登出会调用服务端撤销接口，再清理前端 session。
- 管理员重置成员密码后端/API 会返回一次性临时密码；成员列表更多菜单可触发重置，前端弹窗展示临时密码并尝试复制。
- 管理员重置成员密码生成的临时密码会重试直到满足部署级密码复杂度策略，避免随机密码偶发违反策略导致重置失败。
- 管理员直接重置用户密码会经过密码复杂度策略校验。
- 头像上传路径已统一到后端实际接口。
- 公开注册默认关闭：登录页和注册页默认不暴露普通注册入口，匿名注册 API 默认拒绝。
- 人员档案可以不绑定登录账号，支持先维护组织人员档案。
- 人员列表支持“开通账号”行操作：创建/复用全局用户，创建/复用当前租户成员，绑定人员和成员；新建用户时返回一次性临时密码。
- 租户成员列表具备成员查询、导入、状态流转、移除和管理员重置密码等账号维护基础能力。
- 账号页直接从已有人员新增成员已闭环：`admin:provision_member_from_employee` 绑定 `tenantMemberCommandHandler`，页面提供人员选择弹窗，真浏览器 E2E 验证临时密码弹窗。
- 租户成员离职/停用会触发当前实现下的人员离职标记和 session 撤销，不会禁用全局用户账号。
- 系统管理已提供只读 `账号安全策略` 页，展示当前管理员托管、自助密码关闭、公开注册关闭、复杂度、历史密码、过期和锁定等策略事实。

已验证证据：

- OSS 隔离运行环境：`account-org-settings`，slot `94`，backend `http://127.0.0.1:6494`，Vite `http://127.0.0.1:5194`，BFF `http://127.0.0.1:6194`。
- 运行态登录页无公开注册入口，`/signup` 默认回到 `/login`。
- 运行态侧边栏可进入 `组织管理 -> 人员 / 团队 / 账号`，可进入 `系统管理 -> 模型服务`。
- 运行态人员表单 `系统用户` 字段非必填，人员档案可以先于登录账号创建。
- 运行态 API 和 UI 证明匿名注册默认拒绝、登出后旧 token 无效、管理员重置成员密码返回并展示一次性临时密码、成员离职联动人员档案为 `resigned` 且不禁用全局用户。
- 运行态 UI 证明个人资料编辑、头像上传、团队详情成员列表可用。
- 运行态 UI 证明 `系统管理 -> 账号安全策略` 可见且只读展示当前策略。
- 运行态 UI 证明 `组织管理 -> 人员` 可从人员行触发开通账号，并展示一次性临时密码。

- 注册策略页：默认关闭公开注册，后续支持邀请注册、审批注册、SSO 同步等受控入口。
- 账号安全策略页从只读展示升级为配置页：支持租户级行为开关，并明确哪些密码规则仍为部署级。

- 账号页完整新增成员并绑定人员的向导式增强体验。
- 离职 UI 的非破坏性演示链路；当前已有运行态 API 证据和 UI 确认弹窗截图，未执行真实 UI 离职以避免破坏当前 active 测试成员。
- 角色和权限/授权关系当前共用同一路由，后续应拆成清晰 tab 或合并为单入口。

## 交付边界

本轮客户交付按“基础账号闭环”验收：

- 必须可从菜单进入组织、人员、团队、账号、角色、权限、系统设置和模型服务配置。
- 必须默认关闭匿名公开注册。
- 必须支持新增人员档案，且新增人员不强制开通登录账号。
- 必须支持从已有人员档案受控开通登录账号，并在新建用户时一次性展示临时密码。
- 必须支持租户管理员维护已有成员，包括成员列表、状态流转、成员导入和管理员重置密码。
- 账号页从已有人员开通成员已可纳入交付演示；它和人员页开通账号、成员导入、管理员重置密码共同构成当前受控开户和账号维护闭环。
- 必须支持管理员重置成员密码并展示一次性临时密码。当前已通过更多菜单触发、确认弹窗、临时密码弹窗、网络响应和 E2E 验证。
- 必须提供只读账号安全策略视图，说明当前管理员托管、公开注册关闭、自助密码关闭和部署级密码规则。
- 必须支持登出时服务端撤销当前 session。
- 必须支持成员离职/停用时不误伤跨租户全局账号。
- 必须有运行态截图或 E2E 证据证明菜单、注册关闭、重置密码、登出、人员/成员/团队核心路径可用。

不在本轮强承诺范围，但要给出清晰后续计划：

- 完整租户级账号安全策略配置页。
- 邀请注册、审批注册、SSO 同步。
- 账号页完整新增成员并绑定已有人员的增强体验。
- 重置后强制改密的完整登录引导。
- 租户级 session 分区后的精确 session 撤销。

## 运行与交付注意事项

插件菜单、页面、命令配置变更后，运行环境需要重新导入或刷新插件配置。只修改 `plugins/**/config/*.json` 不代表已运行实例立即生效；交付前必须在目标环境验证菜单、权限和页面路由。
