方案 C · A+B 组合 详解

列表页按钮走 action，详情页审批视图走 bpm-panel block

A 面：action.executionMode = "bpm"

只扩现有 action，不改协议。引擎识别 executionMode: "bpm" 后，不走 Command/Automation，直接走 BpmStartProcessService。

{
  "code": "submit_leave_approval",
  "label": { "zh_CN": "提交审批" },
  "executionMode": "bpm",
  "bpm": {
    "processKey": "leave_request",
    "businessKeyField": "id",
    "variables": { "days": "$.days", "reason": "$.reason" },
    "formBindingRef": "leave_request_form",
    "onSuccess": { "toast": "已提交", "refresh": true }
  },
  "placement": ["list.toolbar", "detail.toolbar"],
  "permission": "leave_request.submit"
}

关键点：

  • businessKeyField：告诉引擎用哪个字段做流程业务主键（同一记录不能重复发起）
  • variables：JSONPath 从当前记录抽流程变量（供规则/网关/SLA 使用）
  • formBindingRef：引用已存在的表单绑定，避免重复配置
  • placement：复用 action 现有的位置语义，列表工具栏/详情工具栏都能放
  • 权限/i18n/审计全部自动继承 action 基建

B 面：blockType = "bpm-panel"

新 blockType，放在详情页，渲染"当前审批状态 + 流程图 + 审批操作区 + 历史"。

{
  "blockType": "bpm-panel",
  "id": "approval",
  "bpm": {
    "processKey": "leave_request",
    "businessKeyField": "id",
    "sections": {
      "status":      { "visible": true, "showAssignees": true, "showSla": true },
      "diagram":     { "visible": true, "highlightActiveNode": true },
      "operations":  {
        "visible": true,
        "operations": ["approve", "reject", "addSign", "transfer", "withdraw", "cc"],
        "approveForm": { "fieldsFromNode": true, "commentRequired": false },
        "rejectForm":  { "commentRequired": true }
      },
      "history":     { "visible": true, "showComments": true, "showAttachments": true }
    }
  },
  "visibleWhen": "record.process_instance_id != null"
}

4 个内置 section（可独立开关）：

  1. status：当前节点、当前审批人头像组、SLA 剩余时间
  2. diagram：BPMN 缩略图 + 高亮当前节点（复用 core-designer 的只读渲染器）
  3. operations：根据当前用户是否为候选审批人 + 权限动态显示按钮组；每种操作背后调用已有的 TaskController endpoint
  4. history：审批历史时间线（审批人/操作/意见/附件），来自 BpmAuditService

可见性：visibleWhen 让尚未发起流程的记录不渲染这块，发起后自动出现。

PropertySchema 新增

  • action.executionMode：新增 "bpm" 枚举值；dependsOn 触发 action.bpm 子对象显示
  • action.bpm：processKey（流程选择器，从 BPM 引擎拉列表）、businessKeyField（字段选择器）、variables（JSONPath 键值对）、formBindingRef（表单绑定选择器）
  • blockType: "bpm-panel" 本身的 Schema：processKey + 4 个 section 各自的开关/子配置
  • 全部走 SchemaBlockConfigPanel 自动渲染（遵守 Studio 硬约束）
