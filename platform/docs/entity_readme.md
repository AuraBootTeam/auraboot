数字标牌系统核心实体职责与关系分析
这五个实体构成了数字标牌系统的核心数据模型，形成了一个完整的内容管理和播放控制体系：

1. 实体职责分析
   Content（内容素材实体）

职责：管理所有媒体素材的基础信息
核心功能：存储图片、视频、文本、HTML、URL等各类内容的元数据
关键字段：文件URL、尺寸、时长、MIME类型、样式配置等
业务价值：作为系统的素材库，为上层业务提供内容资源
Program（节目实体）

职责：定义播放节目的整体配置和规则
核心功能：管理节目的播放策略、时间安排、循环规则
关键字段：节目类型（顺序/循环/定时）、播放时间段、有效期、播放规则
业务价值：作为内容组织的逻辑单元，定义"播什么"和"怎么播"
ProgramContent（节目内容关联实体）

职责：建立节目与具体内容的关联关系
核心功能：定义内容在节目中的播放顺序、时长、区域、动画效果
关键字段：播放顺序、持续时间、区域配置、动画配置
业务价值：实现节目的精细化内容编排
Playlist（播放列表实体）

职责：管理设备级别的播放任务和调度
核心功能：组织多个节目的播放序列，支持优先级和调度配置
关键字段：播放列表类型（普通/紧急/模板）、优先级、调度配置、循环设置
业务价值：作为设备播放的执行单元，解决"什么时候播什么节目"
PlaylistProgram（播放列表节目关联实体）

职责：建立播放列表与节目的关联关系
核心功能：定义节目在播放列表中的顺序、权重、播放限制、时段配置
关键字段：播放顺序、播放权重、播放次数限制、时段配置
业务价值：实现播放列表的灵活调度和智能播放
2. 实体关系图谱
   PlainText



Content (素材库)    ↓ 1:NProgramContent (节目内容关联)    ↓ N:1  Program (节目)    ↓ 1:NPlaylistProgram (播放列表节目关联)    ↓ N:1Playlist (播放列表)
3. 业务流程关系
   内容创建流程：

1.
上传素材 → Content实体
2.
创建节目 → Program实体
3.
关联内容到节目 → ProgramContent实体
4.
创建播放列表 → Playlist实体
5.
关联节目到播放列表 → PlaylistProgram实体
播放执行流程：

1.
设备获取播放列表（Playlist）
2.
按优先级和调度规则选择节目（PlaylistProgram）
3.
按播放顺序获取节目内容（ProgramContent）
4.
渲染播放具体素材（Content）
4. 设计特点
   分层设计：

素材层（Content）：管理原始资源
节目层（Program + ProgramContent）：内容编排
播放层（Playlist + PlaylistProgram）：播放调度
灵活性：

支持多种内容类型和播放模式
支持复杂的时间调度和优先级管理
支持动画效果和区域配置
可扩展性：

采用JSON配置字段存储复杂规则
支持多租户和软删除
统一的审计字段设计
这种设计实现了内容管理的高度灵活性，既能满足简单的轮播需求，也能支持复杂的定时调度和紧急插播场景。
